# 声音大百科 · 开发文档 v4.0

> 本文档是项目的**固定技术设定**——定义架构、规范、工具链和开发流程，不随产品迭代频繁变动。

## 一、项目概述

### 1.1 定位

| 维度 | 内容 |
|------|------|
| 产品名称 | 声音大百科（Sound Encyclopedia） |
| 目标用户 | 1-6 岁儿童（家长陪同操作） |
| 核心价值 | 用真实声音帮助孩子认识自然、动物与世界 |
| 商业模式 | 不考虑商业化，专注开源社区共建 |

### 为什么用单仓库？

- 数据和前端放一起，PR 一次提交就能同时加声音和验证效果
- GitHub Pages 部署简单，一个仓库一个地址
- 未来如果声音库长大了，把 `data/` 拆成独立仓库零成本

## 三、数据规范

### 3.1 categories.json（分类定义，人写）

```json
[
  {
    "id": "animals",
    "name_zh": "动物",
    "name_en": "Animals",
    "emoji": "🐾",
    "color": "#FF6F00",
    "subcategories": [
      { "id": "pet", "name_zh": "宠物", "emoji": "🐕" },
      { "id": "farm", "name_zh": "农场", "emoji": "🏠" }
    ]
  }
]
```

### 3.2 单条声音 meta.json（每个声音一个）

放在 `data/sounds/{category}/{name}/meta.json`：

```json
{
  "id": "animals.cat",
  "category": "animals",
  "subcategory": "pet",
  "name_zh": "小猫",
  "name_en": "Cat",
  "emoji": "🐱",
  "description": "优雅独立的毛绒猎人",
  "sounds": [
    {
      "file": "audio/cat1a.mp3",
      "label": "撒娇",
      "fun_fact": "猫咪撒娇时会发出柔软的喵喵声，想引起主人的注意...",
      "fun_fact_ai_generated": true,
      "tags": ["柔软", "高频"],
      "tts": "generated/track-0-tts.mp3"
    },
    {
      "file": "audio/purring.mp3",
      "label": "呼噜",
      "fun_fact": "猫咪呼噜声表示它很满足，像肚子里藏了一个小马达...",
      "tts": "generated/track-1-tts.mp3"
    }
  ],
  "fun_fact": "小猫的脚底有软软的肉垫，走路静悄悄...",
  "fun_fact_ai_generated": true,
  "ai": {
    "generated_at": "2026-06-30T07:00:00.000Z",
    "model": "deepseek-v4-flash",
    "prompt_version": "v1"
  },
  "tts": {
    "name_zh": "generated/name-zh.mp3",
    "name_en": "generated/name-en.mp3",
    "fun_fact": "generated/fun-fact.mp3",
    "voice_id_zh": "female-tianmei",
    "voice_id_en": "male-qn-qingse",
    "generated_at": "2026-06-30T07:15:00.000Z",
    "tts_model": "speech-2.8-hd"
  },
  "tags": ["宠物", "喵喵", "小猫"],
  "license": "CC0-1.0",
  "source": "https://freesound.org/xxx",
  "contributor": "wbyan2021",
  "added_at": "2026-06-17"
}
```

**字段说明：**

| 字段 | 层级 | 必填 | 说明 |
|------|------|------|------|
| `id` | 声音级 | 是 | 格式 `{category}.{name}`，全局唯一 |
| `category` | 声音级 | 是 | 对应 categories.json 中的分类 id |
| `subcategory` | 声音级 | 否 | 子分类 id |
| `name_zh` | 声音级 | 是 | 中文名 |
| `name_en` | 声音级 | 是 | 英文名（也是目录名） |
| `emoji` | 声音级 | 是 | 前端显示用 |
| `description` | 声音级 | 否 | 一句话描述 |
| `sounds` | 声音级 | 是 | 音频文件数组，至少 1 条 |
| `sounds[].file` | 音频级 | 是 | 相对 meta.json 的路径 |
| `sounds[].label` | 音频级 | 否 | 这个音频的标签（如"撒娇"、"警告"） |
| `sounds[].fun_fact` | 音频级 | 否 | 这个音频的专属科普文案 |
| `sounds[].fun_fact_ai_generated` | 音频级 | 否 | `true` 表示由 AI 生成 |
| `sounds[].tags` | 音频级 | 否 | 这个音频的分类标签 |
| `sounds[].tts` | 音频级 | 否 | 这个音频的专属朗读音频路径 |
| `fun_fact` | 声音级 | 否 | 声音级科普文案（作为音频级文案的 fallback） |
| `fun_fact_ai_generated` | 声音级 | 否 | `true` 表示由 AI 生成 |
| `tts.name_zh` | 声音级 | 否 | AI 朗读中文名音频路径 |
| `tts.name_en` | 声音级 | 否 | AI 朗读英文名音频路径 |
| `tts.fun_fact` | 声音级 | 否 | AI 朗读科普文案音频路径 |
| `tags` | 声音级 | 否 | 搜索标签 |
| `license` | 声音级 | 是 | 必须是 CC0 或可商用授权 |
| `source` | 声音级 | 是 | 音频来源 URL |
| `contributor` | 声音级 | 是 | GitHub 用户名 |
| `added_at` | 声音级 | 是 | 添加日期 |

### 3.3 文案层级与 fallback 逻辑

前端展示文案时的优先级：

```
1. 当前播放音频的专属文案 (sounds[currentTrackIndex].fun_fact)  → 显示 🎵
2. 声音级文案 (fun_fact)                                          → 显示 🤖 (AI) 或 💡
3. 默认文案 ("它是神奇世界里的一员...")                             → 兜底
```

TTS 朗读同理：优先用音频级 TTS，fallback 到声音级 TTS。

### 3.4 manifest.json（脚本自动生成，禁止手改）

```json
{
  "version": "2026.06.30.41",
  "generated_at": "2026-06-30T09:00:00.000Z",
  "total_sounds": 58,
  "total_audio_files": 270,
  "categories": [ ... ],
  "sounds": [ ... ]
}
```

> 注意：build-manifest.js 会检查每个音频文件是否真实存在，不存在的自动过滤（磁盘同步）。

## 四、AI 模型配置

### 4.1 .env 配置结构（LLM / TTS 分离）

```bash
# ---- 通用 ----
MINIMAX_GROUP_ID=your_group_id

# ---- LLM 文本模型（DeepSeek 官方）----
LLM_ENDPOINT=https://api.deepseek.com/chat/completions
LLM_API_KEY=sk-your_deepseek_key
LLM_MODEL=deepseek-v4-flash
LLM_MAX_TOKENS=200
LLM_TEMPERATURE=0.8

# ---- TTS 语音合成模型（MiniMax 国内版）----
TTS_ENDPOINT=https://api.minimaxi.com/v1/t2a_v2
TTS_API_KEY=sk-your_minimax_key
TTS_MODEL=speech-2.8-hd
TTS_VOICE_ZH=female-tianmei
TTS_VOICE_EN=male-qn-qingse
TTS_SAMPLE_RATE=32000
TTS_BITRATE=128000
TTS_FORMAT=mp3
TTS_CHANNEL=2
```

### 4.2 MiniMax 国内版 vs 国际版差异（重要！）

| 项 | 国内版 | 国际版 |
|----|--------|--------|
| 域名 | `api.minimaxi.com` | `api.minimax.io` |
| 模型名 | `speech-2.8-hd`（小写） | `Speech-2.8-HD`（大写） |
| 字段名 | `audio_sample_rate` | `sample_rate` |
| 音频编码 | **hex 字符串** | base64 字符串 |
| voice_setting | 需要 `pitch` | 不需要 |

> `scripts/lib/minimax.js` 已做兼容：hex/base64 自动检测解码。

### 4.3 scripts/lib/minimax.js 核心 API

```js
const { loadEnv, getLLMConfig, getTTSConfig, callLLM, generateSpeech, generateFunFact, generateAndSaveSpeech } = require('./lib/minimax');

loadEnv();                          // 加载 .env
const llmCfg = getLLMConfig();      // { apiKey, endpoint, model, maxTokens, temperature }
const ttsCfg = getTTSConfig();      // { apiKey, endpoint, model, sampleRate, ... }

// LLM 调用
const text = await callLLM(messages, options);

// TTS 调用（返回 MP3 Buffer）
const { audioBuffer, format } = await generateSpeech(text, 'zh-fact');

// 业务方法
const result = await generateFunFact(meta);          // 生成科普文案
const filePath = await generateAndSaveSpeech(text, outputDir, filename, 'zh-fact');
```

## 五、构建脚本

### 5.1 build-manifest.js

**职责**：扫描 `data/sounds/` 下所有 meta.json + 读取 categories.json → 生成 manifest.json
**新增**：检查音频文件是否真实存在，不存在的自动过滤（磁盘同步）

```bash
node scripts/build-manifest.js              # 生成 manifest.json
node scripts/build-manifest.js --dry-run    # 只输出统计不写文件
```

### 5.2 validate.js

校验 data/sounds/ 下的所有数据（meta.json 必填字段、id 格式、音频文件存在等）。

### 5.3 ai-generate-fun-fact.js

批量调用 DeepSeek LLM 为声音生成儿童科普文案。

```bash
node scripts/ai-generate-fun-fact.js                    # 为缺少 fun_fact 的声音生成
node scripts/ai-generate-fun-fact.js --force            # 强制重新生成全部
node scripts/ai-generate-fun-fact.js --id animals.cat   # 只生成指定声音
node scripts/ai-generate-fun-fact.js --category animals # 只生成指定分类
```

### 5.4 ai-generate-tts.js

批量调用 MiniMax TTS 生成 AI 朗读音频（中文名/英文名/科普）。

```bash
node scripts/ai-generate-tts.js                    # 为缺少 TTS 的声音生成
node scripts/ai-generate-tts.js --force            # 强制重新生成
node scripts/ai-generate-tts.js --id animals.cat   # 只生成指定声音
```

**注意**：DELAY_MS = 2500（规避 MiniMax RPM 限流）。

## 六、管理后台（admin-server.js）

### 6.1 启动

```bash
node scripts/admin-server.js
# 浏览器打开 http://localhost:3099（仅 127.0.0.1）
```

### 6.2 API 列表

| 方法 | 路径 | 功能 |
|------|------|------|
| GET | `/api/status` | 数据统计（声音级 + 音频级）+ 配置状态 |
| POST | `/api/run` | 运行脚本（SSE 流式输出） |
| POST | `/api/save-config` | 保存 .env 配置 |
| DELETE | `/api/sound?id=xxx` | 删除声音 |
| POST | `/api/sound` | 创建声音（base64 音频） |
| GET | `/api/sound-detail?id=xxx` | 获取声音详情（含每个音频的 fileUrl/ttsUrl） |
| PUT | `/api/sound-meta` | 更新声音级 + 音频级字段 |
| POST | `/api/track-generate-fact` | 单条音频 LLM 生成文案 |
| POST | `/api/track-generate-tts` | 单条音频生成朗读 |
| PUT | `/api/track-update` | 更新单条音频（label/fun_fact/tags） |
| POST | `/api/sync-disk` | 磁盘同步（清理无效引用） |
| GET | `/data/*` | 静态文件服务（音频/JSON） |

### 6.3 UI 功能

- **配置区**：Group ID + LLM 配置卡片 + TTS 配置卡片
- **统计仪表盘**：6 格（声音总数 / 声音级文案 / 声音级TTS / 音频级文案 / 音频级TTS / 待处理）
- **AI 批量生成**：配置项 + 预览 + 进度条 + 完成后自动刷新
- **声音管理**：列表 + 搜索 + 单条操作（📝/🔊/🗑️）+ 展开详情编辑
- **音频级管理**：每个音频独立卡片（播放器 + label + 文案 + LLM生成 + 标签 + 朗读生成 + TTS播放器）
- **终端输出**：SSE 实时日志

## 七、前端设计

### 7.1 技术栈

| 项 | 选择 | 理由 |
|----|------|------|
| 框架 | 原生 HTML/CSS/JS | 简单、零依赖、易扩展 |
| 设计语言 | Glassmorphism + 动态 CSS 变量 | 现代扁平拟物风格 |
| 字体库 | 系统字体栈 | 零依赖、加载零延迟 |
| 数据 | fetch manifest.json | 数据驱动 |
| 音频 | HTML5 Audio | 原生支持 |
| 离线 | Service Worker (sw.js) | PWA |
| 本地缓存 | localStorage | 收藏列表和探索进度 |

### 7.2 核心交互

- **开始页**：Canvas 动态粒子背景 + Glassmorphism 毛玻璃卡片
- **懒加载**：点击时加载音频
- **分类 tabs**：从 manifest.categories 动态生成
- **声音卡片**：emoji + 名称 + 收藏按钮，点击触发播放 + 动画 + 底部科普面板
- **搜索框**：中文名、英文名、标签实时检索
- **收藏系统**：localStorage 持久化
- **探索进度**：勋章称号系统
- **随机探索**：随机播放
- **底部科普面板**：播放后滑出，**根据当前播放音频动态切换文案**（音频级 → 声音级 fallback）
- **AI 语音朗读**：中文名/英文名/科普讲解三个按钮
- **logo 跳转后台**：左上角 logo 点击新标签页打开管理后台

## 八、离线方案

| 内容类型 | 缓存策略 | 说明 |
|----------|---------|------|
| manifest.json | network-first | 保证新声音能上线 |
| 音频文件 (.mp3) | cache-first | 省流量 |
| index.html / sw.js | network-first | 保证更新能生效 |

## 九、音频规范

| 项 | 标准 |
|----|------|
| 格式 | MP3 |
| 采样率 | 44100 Hz |
| 比特率 | 128 kbps |
| 时长 | 1-10 秒（自然音可到 30 秒） |
| 响度 | -16 LUFS |
| 体积 | 单条 < 300KB |

## 十、AI 策略红线

- **不生成动物/自然声音**（真实声音红线）
- AI TTS 仅用于朗读文字内容，标注「🤖 AI 朗读」
- 所有 AI 生成内容标注「🤖 AI 生成」
- 文案可手动调整，手动编辑后标记 `fun_fact_ai_generated: false`

## 十二、关键决策记录

| 决策 | 选择 | 理由 |
|------|------|------|
| 仓库结构 | 单仓库 | PR 一次搞定 |
| 图片方案 | emoji | 零版权、加载快 |
| 音频格式 | MP3 | 兼容性最好 |
| 部署方式 | GitHub Pages | 免费、自动 |
| LLM 服务 | DeepSeek 官方 | deepseek-v4-flash，价格便宜 |
| TTS 服务 | MiniMax 国内版 | speech-2.8-hd，中文音色丰富 |
| 配置体系 | LLM/TTS 分离 | 各自独立 endpoint/api_key/model |
| TTS 输出 | MP3 | 符合项目音频规范 |
| 文案层级 | 声音级 + 音频级 | 音频级优先，声音级 fallback |
| AI 内容透明度 | 标注「🤖」 | 对家长透明 |
| 版权协议 | CC0（素材）+ MIT（代码） | 最干净 |
