# 第一部分 · 产品战略

## 一、产品定位

### 1.1 一句话定位

> 一个**零成本、可离线、社区共建**的儿童声音百科——孩子在家就能听到真实的大海、奶牛、消防车，在声音里认识世界。

### 1.2 定位画布

| 维度 | 内容 |
|------|------|
| 产品名称 | 声音大百科（Sound Encyclopedia） |
| 目标用户 | 1-6 岁儿童（家长陪同操作） |
| 核心价值 | 用**真实录音**帮助孩子认识自然、动物与世界 |
| 差异化 | 真实声音红线 + AI 驱动的探索体验（猜声音 / 声音故事 / 智能推荐） |
| 商业模式 | 不商业化，专注开源社区共建 |
| 部署形态 | 静态网页（GitHub Pages）+ PWA 离线 |
| 仓库地址 | https://github.com/wbyan2021/sound-encyclopedia |
| 版权协议 | CC0（素材）+ MIT（代码） |

### 1.3 不可动摇的产品红线（AI 策略红线）

1. **真实声音红线**：动物叫声、自然音效**必须**使用真实录音，AI 永不生成任何模拟动物/自然的声音。
2. **AI 只增强、不替代**：AI 服务于探索体验，不替代家长陪伴。
3. **AI 文字朗读**：AI 仅用于朗读文字内容（科普文案、中文名、英文名、场景、故事），使用温和自然音色；朗读音频统一标注「🤖 AI 朗读」。
4. **透明度红线**：所有 AI 生成/朗读的内容一律标注「🤖」，对家长保持透明；文案手动编辑后标记 `fun_fact_ai_generated: false`。
5. **零成本红线**：纯静态部署，不引入需要持续付费的后端服务。

### 2.3 核心缺口

1. **音频级内容稀疏**：96 条真实音频中仅 7 条有专属文案/朗读，多音频物种的「撒娇 / 警告」差异化体验尚未铺开。
2. **社区零启动**：仓库「活」的信号不足，缺种子内容与传播故事。
3. **重复访问价值低**：缺少持续更新的内容钩子（场景、故事、游戏）。

### 3.3 Phase C：社区冷启动 + AI 贡献辅助（P1）

**目标**：从「单机游戏」变成「有人来玩的社区项目」，AI 降低贡献门槛。

| 任务 | 说明 |
|------|------|
| 自填 50 条 | 自然声音 + 交通 + 生活，让仓库看起来「活着」 |
| 找 3 个种子用户 | 幼儿园老师、宝妈群、早教社群 |
| 写传播故事 | 「一个 XX 为了让孩子在家听到大海的声音」——发小红书/即刻/公众号 |
| 降低贡献门槛 | 网页表单上传音频自动生成 PR（不让用户操作 GitHub） |
| **🤖 AI 贡献辅助** | 网友传音频后 AI 自动建议分类、emoji、描述，用户只需确认 |
| **🤖 AI 声音鉴定** | AI 判断音频是否真实声音（非合成）、是否有明显杂音、响度是否合适 |

详细实现见 [十六章](#十六phase-cai-贡献辅助)。

### 3.5 Phase E：AI 互动增强（P1）

| 任务 | 说明 | 依赖 |
|------|------|------|
| **🤖 AI 猜声音游戏** | 随机播声音 → 孩子选答案 → AI 给鼓励反馈。答对有动画奖励，答错 AI 提示线索 | Phase B 搜索 + 收藏完成后 |
| **🤖 AI 声音故事** | 选 5-6 个声音，AI 生成一条连贯的儿童故事（文字 + 自动播放） | Phase B 场景组合完成后 |
| **🤖 AI 声音伙伴** | 虚拟角色（小动物/小精灵），孩子提问，AI 用儿童语言回答并播放声音。纯文本交互 | Phase B 搜索完成后 |
| **🤖 智能探索推荐** | 听完一个声音，底部滑出「你还可能喜欢」推荐。基于语义关联而非用户画像 | Phase B 收藏系统完成后 |

## 四、风险与应对

| 风险 | 应对 |
|------|------|
| 版权纠纷 | 强制要求 source + license 字段，PR 模板勾选确认 |
| 音质参差 | 自动校验格式 + 人工 review 音质 |
| 响度不一致 | 规范要求 -16 LUFS，未来脚本可自动归一化 |
| 社区冷启动 | 先自己填 50+ 条自然声音，做出样子再推广 |
| AI 文案出错 | 全程人工审核 + 「🤖」透明标注 + 问题反馈 Issue |
| AI 成本失控 | text_hash 去重 + 月度预算告警 + 静态优先架构 |
| 维护者流失 | 写可持续性声明 + 维护者交接文档 + 考虑机构合作 |

**为什么用单仓库？**

- 数据和前端放一起，PR 一次提交就能同时加声音和验证效果。
- GitHub Pages 部署简单，一个仓库一个地址。
- 未来如果声音库长大了，把 `data/` 拆成独立仓库零成本。

## 七、数据规范

### 7.1 categories.json（分类定义，人写）

```json
[
  {
    "id": "animals",
    "name_zh": "动物",
    "name_en": "Animals",
    "emoji": "🐾",
    "color": "#FF6F00",
    "subcategories": [
      { "id": "pet", "name_zh": "宠物", "emoji": "🐕" },
      { "id": "farm", "name_zh": "农场", "emoji": "🏠" }
    ]
  }
]
```

### 7.2 单条声音 meta.json（每个声音一个）

放在 `data/sounds/{category}/{name}/meta.json`：

```json
{
  "id": "animals.cat",
  "category": "animals",
  "subcategory": "pet",
  "name_zh": "小猫",
  "name_en": "Cat",
  "emoji": "🐱",
  "description": "优雅独立的毛绒猎人",
  "sounds": [
    {
      "file": "audio/cat1a.mp3",
      "label": "撒娇",
      "fun_fact": "猫咪撒娇时会发出柔软的喵喵声，想引起主人的注意...",
      "fun_fact_ai_generated": true,
      "tags": ["柔软", "高频"],
      "tts": "generated/track-0-tts.mp3"
    },
    {
      "file": "audio/purring.mp3",
      "label": "呼噜",
      "fun_fact": "猫咪呼噜声表示它很满足，像肚子里藏了一个小马达...",
      "tts": "generated/track-1-tts.mp3"
    }
  ],
  "fun_fact": "小猫的脚底有软软的肉垫，走路静悄悄...",
  "fun_fact_ai_generated": true,
  "ai": {
    "generated_at": "2026-06-30T07:00:00.000Z",
    "model": "deepseek-v4-flash",
    "prompt_version": "v1"
  },
  "tts": {
    "name_zh": "generated/name-zh.mp3",
    "name_en": "generated/name-en.mp3",
    "fun_fact": "generated/fun-fact.mp3",
    "voice_id_zh": "female-tianmei",
    "voice_id_en": "male-qn-qingse",
    "generated_at": "2026-06-30T07:15:00.000Z",
    "tts_model": "speech-2.8-hd"
  },
  "tags": ["宠物", "喵喵", "小猫"],
  "license": "CC0-1.0",
  "source": "https://freesound.org/xxx",
  "contributor": "wbyan2021",
  "added_at": "2026-06-17"

```

**字段说明：**

| 字段 | 层级 | 必填 | 说明 |
|------|------|------|------|
| `id` | 声音级 | 是 | 格式 `{category}.{name}`，全局唯一 |
| `category` | 声音级 | 是 | 对应 categories.json 中的分类 id |
| `subcategory` | 声音级 | 否 | 子分类 id |
| `name_zh` | 声音级 | 是 | 中文名 |
| `name_en` | 声音级 | 是 | 英文名（也是目录名） |
| `emoji` | 声音级 | 是 | 前端显示用 |
| `description` | 声音级 | 否 | 一句话描述 |
| `sounds` | 声音级 | 是 | 音频文件数组，至少 1 条 |
| `sounds[].file` | 音频级 | 是 | 相对 meta.json 的路径 |
| `sounds[].label` | 音频级 | 否 | 这个音频的标签（如"撒娇"、"警告"） |
| `sounds[].fun_fact` | 音频级 | 否 | 这个音频的专属科普文案 |
| `sounds[].fun_fact_ai_generated` | 音频级 | 否 | `true` 表示由 AI 生成 |
| `sounds[].tags` | 音频级 | 否 | 这个音频的分类标签 |
| `sounds[].tts` | 音频级 | 否 | 这个音频的专属朗读音频路径 |
| `fun_fact` | 声音级 | 否 | 声音级科普文案（作为音频级文案的 fallback） |
| `fun_fact_ai_generated` | 声音级 | 否 | `true` 表示由 AI 生成 |
| `tts.name_zh` | 声音级 | 否 | AI 朗读中文名音频路径 |
| `tts.name_en` | 声音级 | 否 | AI 朗读英文名音频路径 |
| `tts.fun_fact` | 声音级 | 否 | AI 朗读科普文案音频路径 |
| `tags` | 声音级 | 否 | 搜索标签 |
| `license` | 声音级 | 是 | 必须是 CC0 或可商用授权 |
| `source` | 声音级 | 是 | 音频来源 URL |
| `contributor` | 声音级 | 是 | GitHub 用户名 |
| `added_at` | 声音级 | 是 | 添加日期 |

### 7.3 文案层级与 fallback 逻辑

前端展示文案时的优先级：

```
1. 当前播放音频的专属文案 (sounds[currentTrackIndex].fun_fact)  → 显示 🎵
2. 声音级文案 (fun_fact)                                          → 显示 🤖 (AI) 或 💡
3. 默认文案 ("它是神奇世界里的一员...")                             → 兜底
```

TTS 朗读同理：优先用音频级 TTS，fallback 到声音级 TTS。

## 十五、Phase B：AI 内容赋能

### 15.1 数据模型（AI 相关字段）

字段完整定义见 [§7.2 单条声音 meta.json](#72-单条声音-metajson)。AI 相关字段要点：

- `fun_fact_ai_generated`：标识是否 AI 生成，前端展示「🤖」角标。
- `ai.generated_at`：生成时间，用于判断是否需要重新生成。
- `ai.model` / `ai.prompt_version`：记录模型与 prompt 版本，便于追溯、成本核算与批量优化。
- `ai.text_hash`：文本哈希，避免内容无变化时重复调用 API。
- `tts.*`：AI 朗读音频路径与生成信息；真实声音放 `audio/`，AI 朗读放 `generated/`，避免混淆。

### 15.2 AI 科普文案与描述生成

调用 LLM（DeepSeek `deepseek-v4-flash`，OpenAI 兼容格式）。输入以动物标签为核心（物种、习性、栖息地、分类、子分类），让生成内容更准确、有针对性。

**科普文案 Prompt 模板（v1）：**

```text
你是一位儿童科普作家，擅长为 1-6 岁儿童撰写简短、有趣、准确的科普小知识。
请根据以下动物标签，写一段 30-50 字的中文科普文案。

要求：
- 只陈述科学事实，不编造；
- 语言口语化、有画面感，孩子能听懂；
- 充分利用提供的标签（物种、习性、栖息地等），突出一个有趣的冷知识；
- 不输出任何解释、标题或多余内容，只返回一段文案。

输入：
- 中文名：{name_zh}
- 英文名：{name_en}
- 分类：{category} / {subcategory}
- 标签：{tags}
- 栖息地：{habitat}
- 现有描述：{description}
```

**描述生成 Prompt（可选）：**

```text
请根据以下标签，为声音条目写一段不超过 30 字的简短描述，让孩子和家长一眼就知道这是什么声音。
输入：中文名、英文名、分类、标签。
```

**触发方式：**

| 方式 | 场景 | 实现位置 |
|------|------|---------|
| 本地脚本批量生成 | 维护者本地为全部/指定声音生成文案 | `scripts/ai-generate-fun-fact.js` |
| GitHub Actions 自动补全 | PR 合并后扫描缺失 `fun_fact` 的条目 | `.github/workflows/`（规划中） |
| 管理后台单条生成 | 本地为某个声音/音频重新生成 | `admin-server.js` → `/api/track-generate-fact` |

**审核流程：**

```
AI 生成文案 → 写入临时分支/本地 → 人工 review 修改 → 合并到 main → build-manifest 重新生成 manifest
```

### 15.3 AI 朗读音频（TTS）

调用 MiniMax TTS（国内版 `speech-2.8-hd`）为声音生成朗读音频，覆盖中文名、英文名、科普文案三类，输出 MP3 存入 `generated/`。

- **声音级**：`name-zh.mp3` / `name-en.mp3` / `fun-fact.mp3`，已全量完成 58/58。
- **音频级**：`track-{N}-tts.mp3`，为每条音频的专属文案生成朗读，进行中 7/96。
- **限流**：批量生成时 `DELAY_MS = 2500` 规避 MiniMax RPM 限制。
- **脚本**：`scripts/ai-generate-tts.js`；单条可用管理后台 `/api/track-generate-tts`。

> 中文名/英文名独立播报，与科普文案分开，前端三个朗读按钮各自对应。

### 15.4 AI 场景组合

目标：解决「重复访问价值低、缺少持续内容」的问题。新增 `data/scenes.json`：

```json
{
  "scenes": [
    {
      "id": "forest-morning",
      "name_zh": "森林早晨",
      "emoji": "🌲",
      "description": "清晨的森林，小鸟最先醒来唱歌",
      "sound_ids": ["animals.bird", "nature.stream", "nature.wind", "animals.woodpecker"],
      "ai_composed": true,
      "ai": { "model": "deepseek-v4-flash", "generated_at": "2026-06-29T12:00:00Z" }
    }
  ]
}
```

场景编排 Prompt：

```text
请从以下声音列表中挑选 3-6 个，组合成一个适合 1-6 岁儿童聆听的场景「{场景主题}」。
要求：
- 场景有叙事感，声音按时间或空间顺序排列；
- 只使用列表中存在的 sound_id；
- 返回 JSON 数组，元素为 sound_id。

可用声音列表：
{manifest.sounds 的 id + name_zh + tags}
```

场景随 `manifest.json` 一起被前端读取，首页增加「🎬 场景」入口，进入后按顺序连续播放，每个声音高亮对应卡片。

### 15.5 多音频随机轮播（播放体验增强）

针对拥有多条真实音频的物种条目，每次点击卡片或切换详情时，从 `sounds` 数组中随机抽取一条播放，避免孩子反复听到同一段声音，提升探索新鲜感。

- 前端维护当前播放索引，每次触发播放生成 `Math.floor(Math.random() * sounds.length)`。
- 短时间内重复点击仍可随机到不同音频；连续两次相同不强制去重（简单可靠）。
- 随机逻辑纯前端实现，不依赖 AI；音频列表来自 AI 生成/人工维护的 `meta.json`。
- 与 AI TTS 互不干扰：真实声音随机播放，TTS 朗读固定对应文案。

## 十七、Phase E：AI 互动增强

### 17.1 AI 猜声音游戏

**入口：** 首页顶部「🎮 猜声音」按钮。

**玩法：**
1. 系统从 `manifest.sounds` 中随机选 1 个答案。
2. 播放该声音的真实音频（从其 `sounds` 中随机一条）。
3. 展示 4 个候选卡片（emoji + 中文名），1 个正确答案 + 3 个干扰项。
4. 干扰项选择策略：优先同分类/同子分类/同标签，其次随机。
5. 孩子选择后：
   - 答对：撒花动画 + 随机鼓励文案（如「太棒了！这是小狗，它的鼻子可灵啦！」）。
   - 答错：高亮正确答案 + 播放正确答案音频 + 显示其 `fun_fact` + 温和提示「没关系，再听一次就记住啦」。

**AI 使用点：** 鼓励/提示文案可预生成模板库（20-30 条），也可调用 LLM 动态生成。干扰项首期用规则，未来可升级为 LLM 语义推荐。

**进度与勋章：** 答对计入「猜声音成就」，与现有探索进度系统打通。

### 17.2 智能探索推荐

**入口：** 底部科普面板「✨ 你还可能喜欢」区域。

- 基于当前声音的标签、子分类、分类，计算相似度最高的 3 个声音。
- 相似度算法：共同标签权重 50% + 同子分类权重 30% + 同主分类权重 20%。
- 推荐结果以卡片展示，点击直接跳转并播放。
- 可预生成推荐语（如「小狗和狼都是犬科动物，听听狼怎么叫吧」），写入 `data/recommendations.json` 或前端通过模板生成。

### 17.3 AI 声音伙伴（儿童问答）

**入口：** 底部面板或悬浮按钮「💬 问小精灵」。

- 预生成 20-30 个高频问题的 QA 映射，写入 `data/qa-cache.json`。
- 前端通过关键词匹配用户输入，命中则展示答案和相关声音卡片。
- 未命中时提示「这个问题好有趣，去问问爸爸妈妈吧」。
- 未来可扩展为本地代理调用 LLM 动态回答，首期保持静态部署友好。

### 17.4 AI 声音故事

**入口：** 首页「📖 听故事」模式。

**流程：**
1. 孩子选择 3-5 个声音（或选择系统推荐主题，如「农场的一天」）。
2. 调用 LLM 生成一段连贯的儿童故事，每个声音对应一个情节。
3. 播放时按故事顺序自动播放声音，并高亮对应卡片。
4. 故事文本以纯文字展示在底部面板；同时提供「🔊 AI 读故事」按钮，由 AI 朗读故事文字（不朗读真实声音）。

**故事 Prompt：**

```text
请用以下声音为 1-6 岁儿童编写一段 100-150 字的小故事。
要求：
- 每个声音在故事中出现一次，自然衔接；
- 语言口语化、有画面感；
- 不使用复杂成语；
- 只返回故事正文。

声音列表：
{sound_id + name_zh}
```

故事生成结果可缓存到 `data/stories/` 或 `data/scenes.json`，避免重复调用 LLM。

## 十八、AI 测试与质量保障

| 测试项 | 方法 |
|--------|------|
| AI 文案准确性 | 抽样人工审核，建立「问题文案」反馈 Issue 模板 |
| 儿童友好度 | 邀请 1-2 位家长/老师试读，收集反馈 |
| 生成成本监控 | 记录每次 API 调用的 token/字符数，设置月度预算告警 |
| 内容安全 | 增加敏感词过滤，禁止 AI 生成恐怖、暴力、不适宜儿童的内容 |
| 离线可用性 | 确保 AI 生成内容（fun_fact、场景、故事）被 Service Worker 缓存 |
| 回归测试 | 为 `build-manifest.js` 和 `validate.js` 补充测试用例 |
